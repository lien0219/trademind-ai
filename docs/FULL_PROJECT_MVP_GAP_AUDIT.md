# TradeMind 全项目 MVP 缺口审计（P0–P3）

> **Phase F1**（2026-06-29）— 基于代码与文档的事实审计，按优先级分级。  
> **审计范围**：34 功能模块 + MVP 主链路 16 步。  
> **不在本阶段**：最终人工测试、真实预发、抖店 E2E、生产灰度、打 tag、Production Ready。

## 审计摘要

| 指标 | 数量 |
| --- | --- |
| 模块 `done` | 18 |
| 模块 `partial` | 15 |
| 模块 `missing` / `deferred` | 1（文档中心 Admin 内嵌 — P2） |
| P0 缺口 | 6 |
| P1 缺口 | 14 |
| P2 缺口 | 12 |
| P3 缺口 | 8 |

**结论**：商品采集 → 草稿 → AI → 发布检查 → 刊登批次 **主链路 Demo 可用**；订单/库存/客服/权限/总 Dashboard **需 F2–F6 补齐**；抖店真实 E2E **留 F9**。

---

## P0：主链路断点

> 无页面入口、无数据保存、无失败兜底、无法继续下一步、会越权、会误发布。

| ID | 模块 | 缺口描述 | 影响 | 建议阶段 |
| --- | --- | --- | --- | --- |
| P0-01 | 权限 | 无 RBAC；任意登录管理员全 API 权限 | 多角色试用越权 | F5 |
| P0-02 | 抖店草稿 | 无真实凭证时无法验证 `product.addV2` 线上字段 | 误以为已真实创建平台草稿 | F9（`manual_required` 至 F9） |
| P0-03 | Storage | 本地 storage 无公网 URL，抖店图片上传链路不可端到端 | 刊登主链路在真实抖店场景断点 | F9 + 配置公网 Storage |
| P0-04 | 客服发送 | 平台消息发送失败未统一进失败任务中心 | 发送失败后无全局感知 | F4 |
| P0-05 | 总 Dashboard | 订单/库存/客服异常无总览入口 | 主链路后半段需记忆菜单路径 | F6 |
| P0-06 | 订单详情 | 无独立订单详情页（仅列表 + Tab） | 订单→SKU→扣减跳转链不完整 | F2 |

## P1：影响真实试用

> 状态不清楚、跳转不清楚、错误提示不清楚、失败后无法恢复、配置状态看不懂。

| ID | 模块 | 缺口描述 | 建议阶段 |
| --- | --- | --- | --- |
| P1-01 | 权限 | 无管理员用户管理 UI/API | F5 |
| P1-02 | 配置 | 无统一配置健康/状态中心 | F5 |
| P1-03 | 平台 | 非抖店平台仅 `local_draft_only`，状态需更清晰标注 | F6 |
| P1-04 | 订单同步 | `partial_success` 页级错误在 UI 展示不足 | F2 |
| P1-05 | 订单异常 | 异常工作台与失败任务中心双向深链不完整 | F2 |
| P1-06 | SKU 匹配 | ambiguous 批量处理 UX 弱 | F2 |
| P1-07 | 库存同步 | `inventory_sync_enabled` 默认 off，新手不知需开启 | F3 |
| P1-08 | 库存同步 | SKU 未绑定阻断提示分散在多个 Tab | F3 |
| P1-09 | 客服 | 会话缺订单/商品上下文侧栏 | F4 |
| P1-10 | 客服 | 待回复数未进 Dashboard | F4 + F6 |
| P1-11 | AI 运营 | 旧版 `/ai/batches` 与新版批次并存，入口易混淆 | F6（已部分修复，需彻底隐藏或迁移） |
| P1-12 | API 文档 | `docs/api.md` 部分 legacy 路径（`/stores`、`/ai/chat`）与实现不一致 | F8 |
| P1-13 | Demo 数据 | 缺订单/库存/客服/失败任务样本 | F7 |
| P1-14 | 抖店 | Release Candidate 状态需在刊登/订单/库存页统一 RC 标识 | F6 |

## P2：体验优化

| ID | 模块 | 缺口描述 | 建议阶段 |
| --- | --- | --- | --- |
| P2-01 | OCR | 无独立 OCR 设置菜单（合并在图片 AI 设置） | F5 或保持现状 |
| P2-02 | 商品列表 | 1000+ 条时筛选项可再优化 | F6 |
| P2-03 | 刊登批次 | 大批次（>100 商品）同步编排耗时提示 | F6 |
| P2-04 | 采集 | 1688/PDD/淘宝需手动浏览器登录，首次引导可加强 | F6 |
| P2-05 | AI 图片 | 14/16 试跑 `passed_with_warning`，白底图 Provider 依赖 | F7 |
| P2-06 | 工作台 | 浏览器后退/刷新状态保持 `manual_required` | F9 |
| P2-07 | 多分辨率 | 真实截图验收 `manual_required` | F9 |
| P2-08 | 文档 | 无 Admin 内嵌帮助/文档中心 | deferred |
| P2-09 | 空状态 | 部分订单/库存/客服列表空态引导可统一 | F6 |
| P2-10 | Integrations | 第三方集成总览可读性可增强 | F5 |
| P2-11 | 告警 | Webhook 告警依赖配置，缺配置时静默 | F5 |
| P2-12 | 旧版入口 | 商品列表旧版批量 AI 标识已加，可进一步收敛 | F6 |
