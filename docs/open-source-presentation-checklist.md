# 开源展示发布前自检清单

这份清单用于在 **对外展示更新前** 快速检查 GitHub 首页、About、截图素材和文档入口是否一致。

适用场景：

- 修改 `README.md` / `README.en.md`
- 替换头图、截图或 Social Preview
- 更新 GitHub `About`、`Topics`
- 调整文档中心首页

建议用法：

- 小改动时，发布前 2 分钟快速过一遍
- 大改动时，与 [task-checklist.md](task-checklist.md) 一起使用

## 首屏检查

- [ ] 项目名、一句话定位和头图表达的是同一个主故事。
- [ ] 首屏没有出现空占位表格、无意义目录或内部阶段噪音。
- [ ] README 顶部徽章不超过“少量关键识别信息”的范围。
- [ ] 中英文 README 结构一致，且都能独立成立。
- [ ] README 第一屏优先展示“项目是什么”，而不是“维护细节是什么”。

## 文案检查

- [ ] `About` 文案足够短，第一次访问的人能立即看懂。
- [ ] 没有把内部术语、里程碑编号或验收状态直接暴露给普通访客。
- [ ] 没有使用空泛营销词，如 `best`、`next generation`、`powerful`。
- [ ] 英文文案自然、紧凑，不是逐字硬翻。
- [ ] README、docs 首页、GitHub About 的定位表述一致。

## 素材检查

- [ ] 头图使用当前最新项目口径。
- [ ] Social Preview 已更新为最新版本。
- [ ] 截图优先展示“已经做成的主线能力”，而不是配置页或半成品。
- [ ] 图片中文案没有超出卡片或出现明显裁切问题。
- [ ] 新素材已加入仓库且路径可访问。

## GitHub 设置检查

- [ ] `About` 已填写推荐描述。
- [ ] `Topics` 已更新，且没有无关或误导性标签。
- [ ] `Social Preview` 已上传最新图片。
- [ ] 默认分支与对外展示分支一致。
- [ ] 仓库首页可直接看到最新 README 展示效果。

## 文档入口检查

- [ ] 新增展示规范或素材脚本后，已加入 [docs/README.md](README.md)。
- [ ] 相关操作说明已沉淀到 [github-repo-presentation.md](github-repo-presentation.md)。
- [ ] 若新增素材生成方式，已说明如何重新生成。
- [ ] 如果展示口径变化，README 与 docs 首页已一起更新。

## 最后确认

- [ ] 把仓库分享给一个不了解项目的人，他能在 10 秒内知道项目大致做什么。
- [ ] 首屏的下一步入口是清晰的：快速开始、文档中心、Roadmap 至少有一个明显入口。
- [ ] 若本次只改展示素材或文案，最终说明中已写明未运行构建/测试的原因。
