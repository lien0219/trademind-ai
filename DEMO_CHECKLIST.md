# TradeMind 演示验收清单（Demo Checklist）

> 用于本地 / 预发环境快速验收核心 MVP 能力。按模块勾选。

## 淘宝/天猫采集器（测试中，2026-06-06）

- [ ] 采集中心卡片显示 **淘宝/天猫采集器**，状态 **测试中**
- [ ] 卡片说明含标题、价格、主图、详情图、商品参数、商品规格
- [ ] 批量采集按钮 **禁用**，提示「淘宝/天猫批量采集暂未开放，请先使用单个采集」
- [ ] `item.taobao.com` 链接识别为 `taobao_tmall`（非 custom）
- [ ] `detail.tmall.com` / `detail.tmall.hk` / `chaoshi.tmall.com` / `ju.taobao.com` 识别为 `taobao_tmall`
- [ ] 淘宝店铺页/搜索页提交时提示 **UNSUPPORTED_TAOBAO_URL**
- [ ] 未登录时任务失败或提示 **LOGIN_REQUIRED**（小白可读）
- [ ] 需要安全验证时提示 **VERIFY_REQUIRED**，引导在采集浏览器完成验证
- [ ] 商品下架时提示 **ITEM_NOT_FOUND**
- [ ] 登录后普通商品能采集 **标题、价格、至少 1 张主图**
- [ ] 主图为空时任务失败 **MAIN_IMAGES_EMPTY**
- [ ] 详情图为空时不阻断草稿，可有 **DETAIL_IMAGES_INCOMPLETE** 提示
- [ ] SKU 不完整时草稿可创建，提示 **SKU_INCOMPLETE**
- [ ] 价格缺失时草稿可创建，提示 **PRICE_NOT_FOUND**，发布前检查拦截
- [ ] 采集成功创建商品草稿：`source=taobao_tmall`、`currency=CNY`、`status=draft`
- [ ] 商品详情页顶部显示淘宝/天猫来源提示
- [ ] 商品草稿 **图片管理** 可「同步图片到平台存储」/ 批量同步主图 / 详情图
- [ ] 失败任务中心展示平台 `taobao_tmall`、失败原因、打开链接、打开采集浏览器、重试、生成告警
- [ ] 设置 → 采集服务 → 淘宝/天猫专属配置（登录状态、打开浏览器、超时、访问检测、重试、滚动等待、详情图等待、SKU 点击）
- [ ] 独立浏览器 Profile `taobao_tmall`（不与 1688/拼多多/custom 混用）
- [ ] 操作日志含创建任务、登录检测、采集成功/失败记录

### 演示流程（建议顺序）

1. **设置**：进入「设置 → 采集服务 → 淘宝/天猫」，点击「打开淘宝/天猫采集浏览器」完成登录，点「重新检测」确认已登录。
2. **单采**：采集中心 → 淘宝/天猫采集器 → 开始采集 → 粘贴 `item.taobao.com` 或 `detail.tmall.com` 商品链接 → 提交。
3. **草稿**：任务成功后打开商品草稿，核对标题、价格、主图、规格；查看顶部采集风险提示。
4. **图片同步**（可选）：图片管理 →「同步图片到平台存储」。
5. **发布检查**：点击发布前检查，确认缺价/缺主图/外链图片等提示。
6. **失败回归**：用未登录态或已下架链接各测一条，在失败任务中心查看原因与重试。

详细 20 条链接验收表见 [`docs/collector-taobao-tmall-test-links.md`](docs/collector-taobao-tmall-test-links.md)。
